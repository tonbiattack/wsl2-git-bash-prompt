## 項目別評価

### 1. git-prompt の読み込み

```bash
source /usr/lib/git-core/git-sh-prompt 2>/dev/null || source /etc/bash_completion.d/git-prompt 2>/dev/null
```

| 観点 | 評価 |
|------|------|
| 動作 | 正常 |
| 安全性 | 良好 |

- メインパスが存在しない場合に fallback パスを試みる設計になっており、環境差異に強い
- `2>/dev/null` でエラー出力を抑制しているため、ファイルが存在しない場合でもエラーが出ない
- どちらのパスにも存在しない場合は `__git_ps1` が未定義になるが、PS1 内の `$()` は空文字になるだけでクラッシュしない

---

### 2. Git 状態表示の変数

```bash
GIT_PS1_SHOWDIRTYSTATE=1
GIT_PS1_SHOWUNTRACKEDFILES=1
```

| 変数 | 効果 |
|------|------|
| `GIT_PS1_SHOWDIRTYSTATE` | 未コミットの変更があると `*`、ステージ済みの変更があると `+` を表示 |
| `GIT_PS1_SHOWUNTRACKEDFILES` | 未追跡ファイルがあると `%` を表示 |

- Git の作業状態を一目で把握できる
- ブランチ名だけでなく変更の有無も確認できるため、コミット忘れの防止に有効

---

### 3. PS1 プロンプト定義

```bash
export PS1='\[\e[32m\]\u@\h\[\e[0m\] \[\e[33m\]\w\[\e[0m\]$(__git_ps1 " \[\e[34m\](%s)\[\e[0m\]")\n\$ '
```

#### 構成要素

| 要素 | 内容 | 色 |
|------|------|----|
| `\u@\h` | ユーザー名@ホスト名 | 緑 (`\e[32m`) |
| `\w` | カレントディレクトリ | 黄色 (`\e[33m`) |
| `(%s)` | ブランチ名 | 青 (`\e[34m`) |
| `\n\$` | 改行 + プロンプト記号 | デフォルト |

#### 評価

| 観点 | 評価 | 備考 |
|------|------|------|
| 色の実装 | 良好 | ANSI エスケープコードを正しく使用 |
| `\[ \]` の使用 | 正しい | 幅ゼロ文字の明示によりカーソルずれを防止 |
| 色リセット | 適切 | 各色付け後に `\e[0m` でリセットされている |
| ブランチ表示 | 適切 | `__git_ps1` の引数内に色コードを埋め込む安全な書き方 |
| Git 外での動作 | 正常 | git リポジトリ外ではブランチ部分が非表示になる |
